# Правила создания карточек

Эти правила используются при переносе уроков с learn.javascript.ru в карточки курса.

## Базовый порядок

1. Открыть страницу конкретного урока.
2. Найти блок "Навигация по уроку" в левом меню или в структуре страницы.
3. Использовать пункты "Навигации по уроку" как обязательную структуру карточек.
4. Один пункт навигации = одна базовая карточка.
5. Поле `term` должно совпадать с заголовком пункта навигации или быть максимально близким к нему.
6. Поле `definition` должно быть коротким сжатием содержания именно соответствующего раздела.
7. `definition` можно переформулировать, но нельзя добавлять смысл, которого нет в этом разделе источника.
8. Поле `hint` использовать только для ключевой подсказки, важной оговорки или короткого мини-фрагмента кода.
9. Не удалять пункты навигации, даже если они кажутся второстепенными.
10. Не добавлять отдельные карточки по деталям, если у них нет собственного пункта в навигации урока.
11. Важные детали внутри раздела, например `break` или `default`, включать в `definition` или `hint` карточки соответствующего заголовка.

## Примеры кода

1. Большие примеры кода не копировать целиком в карточку.
2. Если раздел содержит большой пример, передавать в карточке вывод, идею или поведение, которое демонстрирует пример.
3. Если без кода смысл теряется, добавлять только минимальный ключевой фрагмент.
4. Если пункт навигации называется "Синтаксис" или объясняет форму языковой конструкции, карточка обязана содержать минимальный кодовый шаблон в поле `code`.
5. Нельзя заменять синтаксис только текстовым описанием.
6. Кодовый шаблон должен быть коротким и обобщённым, без лишнего контекста из большого примера.
7. Если ученик после карточки должен уметь написать конструкцию сам, в карточке должен быть пример формы этой конструкции.
8. Если пример содержит несколько самостоятельных идей и кажется, что нужны дополнительные карточки, не дробить его автоматически. Сначала предложить дополнительные карточки отдельно.
9. Горизонтальная прокрутка в блоке `code` допустима, если она сохраняет читаемость кода и не становится основным способом изучения карточки.
10. Код в карточке должен быть коротким: ориентир — до 30 строк.
11. Если пример больше 30 строк, не переносить его целиком. Нужно сократить его до синтаксического скелета, ключевого фрагмента или описать поведение примера в `definition`.
12. Если пример невозможно сократить без потери смысла, сначала предложить отдельный формат для такого материала.

## Картинки

1. Если раздел содержит картинку, не добавлять её автоматически.
2. Картинку можно добавить только если она объясняет ключевую идею раздела, а не является декоративной.
3. Перед добавлением картинки проверить, что UI карточек поддерживает `imageURL` и `imageAlt`.
4. Даже если карточка содержит картинку, `definition` всё равно должен объяснять смысл текстом.

## Перед внесением изменений

Перед изменением кода обязательно показать список в формате:

```text
заголовок источника → будущая карточка
```

Изменения в код вносятся только после подтверждения этого списка.
